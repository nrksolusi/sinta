## Objective

Give document lists a tabbed status/fulfillment view plus an explicit filter bar,
both driving the read-query contract through one shared filter model.

## Required reading (weighted)

- **Must:** `../../plans/fix-4-ui-ux-revamp.md` Slice 4 + Ground-truth ("no `ui/tabs.tsx`",
  `DocList` is controlled with URL state, only removal chips exist); ADR-0019.

## Scope fence

**In scope** - generate `ui/tabs.tsx`; per-doc-type tab presets as named filters
(orders: Aktif / Selesai / Semua, Selesai = fulfillment `closed`); a filter bar
(status/warehouse/date/`q`) above `DocList`. Tabs + bar share one filter model
wired to the existing URL params.

**Out of scope** - server changes (consume SN-0002/04 only); the picker rebuild
(SN-0003 already did it). If a needed filter field isn't in the contract, STOP.
