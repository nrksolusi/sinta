## Objective

Move document-list filtering and entity search off the client and onto the
database, using the contract frozen in SN-0001.

## Required reading (weighted)

- **Must:** ADR-0019; `../../plans/fix-4-ui-ux-revamp.md` Slice 1 + the Ground-truth notes
  on `documents.sql` / in-memory client filtering.
- **Should:** existing `queries/documents.sql` `List*` shapes; catalog
  `status`/`role` filters as the precedent.

## Scope fence

**In scope** - `queries/documents.sql` + list handlers: accept
`status/warehouseId/dateFrom/dateTo/q/cursor/limit`, keyset pagination, return
`nextCursor`. Add `q` typeahead to products/partners/warehouses over the trigram
indexes with match ranking. Handler + domain tests.

**Out of scope** - any client change (SN-0003); tabs/filter presets (SN-0008);
fulfillment fields (SN-0004). Do not alter the OpenAPI shape - if a param is
missing, STOP and amend SN-0001.

## Acceptance gate

- [ ] Every document `List*` honours all filter params; absent params = no filter
- [ ] Cursor pagination deterministic and stable at boundaries (no dupes/skips); `nextCursor` returned
- [ ] `q` typeahead on products/partners/warehouses returns trigram-ranked matches
- [ ] Tests: filter combinations, cursor boundary stability, search ranking (test-first)
- [ ] `go build/vet/test ./...` green; board exit gate satisfied

### 2026-07-23 · implementation done (commit c87777a)
Implemented server-side filtering, cursor pagination, and entity search.

Key decisions:
- Cursor: base64url(json({ts, id})) using (created_at DESC, id DESC) keyset.
  Decoded in `decodeCursor` helper in documents.go; encoded by `encodeCursor`.
  `nextCursorIfMore` generic helper trims rows to page_limit and encodes last row.
- SQL: all 7 doc List* queries updated with full filter set (status, warehouse_id,
  date_from, date_to, q on doc_number, cursor keyset). LIMIT is always passed
  as page_limit + 1 to detect next page.
- Catalog (products, partners, warehouses): q param added to catalog.yaml;
  regenerated. SQL uses ILIKE '%q%' for filtering + -GREATEST(similarity(...))
  for trigram ranking when q is provided. Works with existing pg_trgm GIN indexes
  from migration 0303.
- `resolveDocListFilter` centralises API param -> pgtype conversion. Each handler
  calls it; the stock transfer handler passes it to the `from/to_warehouse_id OR`
  variant in SQL.
- `pgTextFrom` helper added for optional *string -> pgtype.Text.

Tests (all red-first, all green after):

- TestPOListFilterByWarehouse
- TestPOListCursorPagination (no duplicates across pages, correct nextCursor)
- TestProductSearchByQ
