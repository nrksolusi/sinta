## Objective

Move PO/SO fulfillment rollup from client-side join to server-side computation.
Server returns `receivedQty`/`deliveredQty` per line and document-level
`fulfillmentState` (`open|partial|closed`). Posting a receipt/delivery is guarded
so cumulative fulfilled qty cannot exceed ordered qty beyond the tenant tolerance
(default 0). Receipt line cost defaults from the linked PO line's agreed price.

## References

- ADR-0016: order fulfillment is server-computed, with tolerance-bounded over-receipt
- ADR-0017: receiving cost defaults from the PO line
- ADR-0019: read-query contract (fix-4 Slice 1, already in spec)

## Scope fence

**In scope**
- Migration: computed/rollup view or SQL for received/delivered qty per line
- OpenAPI: `receivedQty`, `deliveredQty`, `fulfillmentState` already added to spec in SN-0013; confirm they serialize
- Server handlers: rollup query on GET /purchase-orders/{id} and GET /sales-orders/{id}; list endpoints include rollup; posting guard in POST /goods-receipts and POST /deliveries
- `tenant_settings` already created in migration 0303; tolerance read in posting tx
- Receipt cost default: pre-fill `unitCost` from linked PO line on GET receipt form; permission check on override
- Client: drop `deliveredByLine`/`lineFulfillment`/`receivedByLine` client-side joins; consume server fields

**Out of scope**
- Approval gate (ADR-0015), cancel implementation (SN-0013 stub only)
- Per-product tolerance override (deferred in ADR-0016)
- ADR-0017 audit log table (recorded field only, not full audit trail)

## Acceptance gate

- [ ] GET /purchase-orders/{id} returns `receivedQty` and `fulfillmentState` per line
- [ ] GET /sales-orders/{id} returns `deliveredQty` and `fulfillmentState` per line
- [ ] POST /goods-receipts returns 422 when receipt would exceed ordered qty + tolerance
- [ ] `tenant_settings.tolerance_over_receipt` is read (not hard-coded)
- [ ] Receipt line cost defaults from linked PO line when creating a receipt
- [ ] Client drops client-side joins; uses server rollup fields
