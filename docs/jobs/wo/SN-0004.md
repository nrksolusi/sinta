## Objective

Make the server the source of truth for order fulfillment and enforce the
receiving rules the client currently only approximates.

## Required reading (weighted)

- **Must:** ADR-0016, ADR-0017; `../../plans/fix-4-ui-ux-revamp.md` Slice 2 + Ground-truth
  on `goods_receipts.go:127-151` (posting validates only `qty>0`), the browser
  rollup in `-order-data.ts` / `-sales-order-data.ts`, `posting.go:120-159`.
- **Should:** ADR-0001/0003 (journal append-only - fulfillment is *computed*, not
  a new mutable column); SN-0001's `tenant_settings`.

## Scope fence

**In scope**
- Rollup query: `receivedQty`/`deliveredQty` per PO/SO line + `fulfillmentState`
  (`open|partial|closed`) on line + document responses, computed from posted
  linked receipt/delivery lines.
- Over-receipt guard inside the posting tx, under a per-order-line advisory lock;
  blocks beyond tenant tolerance (default 0 from `tenant_settings`); closed lines
  can't re-receive.
- Receipt line `unitCost` defaults from the PO line; override gated to owner/admin
  and recorded.

**Out of scope** - cancellation (SN-0006, though it reuses this guard); any client
change (SN-0005); do not add a mutable `received_qty` column (compute it); do not
touch `stock_movements` write path beyond the guard.

## Acceptance gate

- [ ] PO/SO line + doc responses expose computed `receivedQty`/`deliveredQty` + `fulfillmentState`
- [ ] Over-receipt beyond tolerance blocked in the posting tx; within tolerance succeeds; boundary tested
- [ ] Concurrent receipts on one order line serialize via advisory lock (race test passes)
- [ ] Closed lines cannot be re-received
- [ ] Receipt `unitCost` defaults from PO line; override rejected for non-owner/admin, recorded when allowed
- [ ] Tests test-first; `go build/vet/test ./...` green; board exit gate satisfied
