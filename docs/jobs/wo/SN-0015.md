## Objective

Let a tenant require approval before a document can be posted, opt-in per document
type. A gated document moves `draft → pending_approval (Menunggu Persetujuan) →
approved (Disetujui) → posted`; approval authorizes, posting stays the separate
step that writes the journal and assigns the number (ADR-0010 unchanged).
Owner/admin approve (no new role); self-approval is allowed; reject returns the
document to `draft` with a required reason. Non-gated document types keep today's
`draft → posted` flow untouched.

## Scope fence

**In scope**
- Migration: drop/re-add each gated header's `status` CHECK to add
  `pending_approval` and `approved` (pattern of `0300`); add
  `approval_settings (tenant_id, doc_type, requires_approval)` with `tenant_id` +
  RLS (ADR-0004); add `submitted_at/by`, `approved_at/by`, `rejected_at/by`,
  `reject_reason` to gated headers (fold with INC-2's columns). Renumber to the
  next free migration after the latest on `main`.
- Spec + server (spec-first: `openapi.yaml` + `paths/*`, then `go generate ./...`):
  `submit` / `approve` / `reject` transitions per document type; posting
  precondition - if the tenant gates this `doc_type`, require `status ==
  approved`, else `status == draft` as today. Reuse `requireDocumentWriter`;
  approve/reject require owner/admin. `approval_settings` get/update endpoints.
- Client: wire the dormant `pending` StatusBadge, add an `approved` ("Disetujui")
  variant + message key; submit/approve/reject actions on draft/pending detail
  views with a reject-reason `ConfirmDialog`; approval events in the UX-D7
  timeline; a settings screen toggling `requires_approval` per doc type
  (owner/admin only).
- Test-first (/tdd): transition guards, posting precondition (gated vs non-gated),
  role checks, reject-to-draft with reason, self-approval allowed.

**Out of scope** - STOP and escalate if the work needs it
- Numbering changes (ADR-0010 stays: number assigned at posting, not approval).
- A dedicated approver role (ADR-0005/0015 keep the fixed role set).
- Enforced separation of duties / blocking self-approval (ADR-0015 allows it).
- Slice 3 (surat jalan printing + numbering config UI) - separate job.
- Any generated file hand-edit; any change to a document type the tenant does not
  gate (its flow must stay byte-for-byte today's).
