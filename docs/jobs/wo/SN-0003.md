## Objective

Make document lists and entity pickers drive the server params from SN-0002, so
filtering/pagination/search happen server-side instead of in the browser.

## Required reading (weighted)

- **Must:** ADR-0019; `../../plans/fix-4-ui-ux-revamp.md` Slice 1 + Ground-truth on
  `DocList`, `combobox.tsx`/`product-combobox.tsx`/`pickers-data.ts`, the
  `onSearch` seam.
- **Should:** `hooks/use-mobile.ts` (Sheet swap for the picker on mobile).

## Scope fence

**In scope** - `DocList` consumers pass filters/cursor to the server via the
existing URL search state; drop in-memory full-list filtering. Rebuild pickers as
a Command **dialog** (Sheet on mobile) driving server `q` through `onSearch`;
preserve recents.

**Out of scope** - `ui/tabs.tsx` + tab presets + filter bar (SN-0008); any server
change. Consume only fields SN-0001/02 provide - if one is missing, STOP.

## Acceptance gate

- [ ] `DocList` consumers server-page and server-filter; no in-memory full-list filtering remains
- [ ] Pickers are a Command dialog (Sheet on mobile) driving server `q`; recents preserved
- [ ] A large list paginates without loading the whole set
- [ ] Tests cover the list-params wiring and picker search (test-first)
- [ ] `pnpm typecheck && pnpm test && pnpm lint` green; `generate-i18n` clean if messages changed; board exit gate satisfied
