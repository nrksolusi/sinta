## Objective

Freeze the API contract and land the shared schema that every fix-4 WO builds on,
so the two lanes can then fork without colliding. No user-facing behaviour ships
in this WO.

## Required reading (weighted)

- **Must:** `../../plans/fix-4-ui-ux-revamp.md` (Slices + Ground truth), ADR-0016/0018/0019,
  `../../plans/m2-m3-roadmap.md` §"Shared schema".
- **Should:** CLAUDE.md spec-first + RLS rules; `../../adr/0010-*` (numbering
  stays at posting - unchanged here).

## Entry gate (WO-specific precondition)

Resolve the `tenant_settings` decision before coding (roadmap open decision #1):
either write the `tenant_settings` ADR, or stub the two settings it backs
(over-receipt tolerance = 0, receiving-cost authority = owner/admin) and file a
follow-up. If fix-3 is running concurrently, coordinate one status-CHECK migration
covering both fix-3 (`pending_approval`/`approved`) and this `cancelled`.

## Scope fence

**In scope**
- `openapi.yaml` + `api/paths/*`: read-query params (`status/warehouseId/dateFrom/
  dateTo/q/cursor/limit`) + `nextCursor` on document `List*`; `q` on products/
  partners/warehouses; fulfillment read fields (`receivedQty`/`deliveredQty`/
  `fulfillmentState`) on PO/SO line + doc; receipt-line `unitCost` default/override
  echo; `cancel` transition (body `reason`) + `cancelled` status on PO/SO.
- Migrations: extend PO/SO `status` CHECK with `cancelled` (drop/re-add per table);
  `tenant_settings` table (`tenant_id` + RLS); `pg_trgm` + trigram indexes on
  searchable name columns; supporting indexes on receipt/delivery -> order-line FKs.
- Regenerate: `go generate ./...`, `pnpm generate-api`; commit generated output.

**Out of scope** - any handler logic, any query beyond schema, any client screen.
Those are SN-0002..11. Do not implement behaviour here.
